## Not Mainnet Ready Yet

The following items still require real external artifacts or credentials before this can be called production deployed:

- Groth16 verifier integration: generated verifier code or verifier program, final verifying key, and audited circuit artifacts for `circuits/credit_score.circom`.
- ZK witness pipeline: production wallet/model commitments must be generated by the same Poseidon encoding used in the circuit.
- Ika CPI: final Ika program ID, instruction interface, account layout, signer rules, and devnet/mainnet deployed addresses.
- Kamino CPI: final Kamino market/vault addresses, CPI account list, authority model, and integration tests against the chosen cluster.
- Deployment authority: program keypair, upgrade authority policy, multisig or hardware wallet process, and environment-specific `AXIOM_PROGRAM_ID`.
- RPC/indexing: QuickNode RPC and WebSocket endpoints with rate limits sized for production traffic.
- Data APIs: GoldRush and Birdeye API keys plus endpoint mapping for all model features and liquidation monitors.
- Torque MCP: production endpoint/auth and campaign execution limits.
- Backend operations: Postgres, Redis, worker process, secret management, monitoring, alerts, and incident runbooks.
- Security review: third-party audit for the Anchor program, ZK circuit, proof generation, and agent transaction policies.
